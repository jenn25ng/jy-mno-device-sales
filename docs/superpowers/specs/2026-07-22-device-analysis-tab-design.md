# 단말별 분석 탭 (Device Analysis Tab) — 설계 문서

- 작성일: 2026-07-22
- 대상 앱: `mno-device-sales` (FastAPI + 단일 HTML SPA, Polaris Data Gateway 메모리 캐시)
- 요청: 현업. "본부별 분석"의 축을 뒤집은 **단말군 중심** 분석 탭 신규 추가.

## 1. 목적 / 한 줄 정의

단말군 하나를 선택하면 그 단말군을 **10개 본부로 분해**해서 포트폴리오·일자별 추이·본부내비중·본부간점유비를 보여주는 탭. 기존 **본부별 분석 탭의 대칭(transpose)** — 본부별은 "본부 → 단말군", 단말별은 "단말군 → 본부".

## 2. 범위 / 결정사항

- 탭 순서: 전사개요 · 본부별분석 · **단말별분석** · 본부매트릭스 · 알림 (본부별 분석 바로 뒤 — 대칭 쌍 인접. ※ 초기 설계는 매트릭스 뒤였으나 2026-07-23 본부별 분석 뒤로 이동)
- 단말군 셀렉터: **11종 전체**(GORDER 순: S26→S25→IP17→IP16→폴더블7→퀀텀6→와이드→A17/16→스타일폴더2→SIMonly→기타). 기본 선택 **S26**.
- 본부 선택 단계 없음 — 항상 10개 본부 동시 표시.
- 전역 필터 **전부 연동**: 기간 · 가입유형 · 판매채널 · 약정유형 · 시점비교. (다른 탭과 동일)
- 섹션: 목업 그대로 전부(아래 4장).
- 접근 방식: **A안** — `build_brief`에 `by_group` 블록 추가(기존 `by_hq` 대칭), 프론트는 칩 클릭 시 클라이언트에서 즉시 전환. 새 엔드포인트 없음.

## 3. 백엔드 설계 — `build_brief`에 `by_group` 추가

기존 `by_hq`의 대칭 구조. 전역 필터·비교(`_resolve_compare`)는 `build_brief`가 이미 배선하고 있어 그대로 재사용. 기간 슬라이스(`tdf`)에서 계산.

### 3.1 응답 구조

```
by_group: [                         # GORDER 순, 11종
  {
    group: "S26",                   # canonical 코드
    total: 16185,                   # 전사 총판매(기간·필터 반영)
    company_share: 17.2,            # 전사비중 = group total / 전체단말군합 (%)
    by_hq: [                        # 10개 본부, count desc 정렬(→ KPI Top1~3)
      { hq: "수도권",
        count: 4638,
        share_of_group: 28.7,       # 본부간점유비(기간 가중) = 본부 G합 / G 전사합
        share_in_hq: 16.2,          # 본부내비중(기간 가중)   = 본부 G합 / 본부 전체판매합
        daily: {"20260601": 134, ...},           # 일자별 판매량
        daily_share_in_hq: {"20260601": 16.2, ...},    # 본부내비중 추이(%)
        daily_share_of_group: {"20260601": 27.7, ...}  # 본부간점유비 추이(%)
      }, ... ],
    daily_total: {"20260601": 480, ...},   # 일자별 판매량 분포(막대) = G 전사 일별합
    total_delta: { abs: 39, pct: 6.3 },    # 비교 하이라이트: G 총판매 증감
    movers: { up: {hq:"PS&M", abs:20}, down: {hq:"제휴", abs:-1} }
  }, ...
]
```

### 3.2 계산 (pandas, 기간 슬라이스 `tdf`)

- `piv = tdf.pivot_table(index=device_group, columns=[mkt_div_org_nm, exec_dt], values=sales_cnt, aggfunc=sum)` 개념. 실제로는 3개 피벗 재사용:
  - **G×HQ×date 판매량** (daily, count) — 본부간점유비 분자 / daily / daily_total(=G×date 합)
  - **HQ×date 전체판매**(모든 단말군 합) — 본부내비중 분모
  - **G×date 전사합**(모든 본부 합) — 본부간점유비 분모
- **본부내비중(j,d)** = `Gjd / HQ_all_jd × 100`
- **본부간점유비(j,d)** = `Gjd / G_total_d × 100`
- **기간 집계는 가중평균**(단순평균 아님):
  - 기간 본부내비중 = `Σd Gjd / Σd HQ_all_jd × 100`
  - 기간 본부간점유비 = `Σd Gjd / Σd G_total_d × 100`
  - 기간 판매량 = `Σd Gjd`
- **비교**: `_resolve_compare`로 비교기간 슬라이스 → 같은 방식으로 G total / 본부별 count 산출 → `total_delta`, `movers`(본부별 증감 최대/최소).
- 0 division 방어(분모 0 → 0.0), 빈 단말군도 항목 유지(빈 시리즈).

### 3.3 build_overview 무관

단말별 탭은 `/api/brief`(전역 기간·compare·channel·agree·scrb_type)만 사용. `build_overview`(전사개요 전용)는 변경 없음. `build_brief` 시그니처에 이미 있는 `compare_to/compare_start/compare_end/channel/agree_type/scrb_type` 재사용.

## 4. 프론트엔드 설계 (`frontend/index.html`)

기존 본부별 분석 탭 컴포넌트(도넛·라인차트·delta칩·sortByG)를 미러링. 색은 본부색(`HQPAL`) 사용(라인/도넛 slice = 본부 단위). 단말군 색(`GCOLOR`)은 셀렉터 칩에만.

### 4-1. 단말군 셀렉터 (유일한 로컬 셀렉터)
- 11종 pill 버튼, 각 pill에 전사 총판매 배지. 기본 **S26**. 본부 선택 단계 없음.

### 4-2. KPI 카드 4개 (가로 1행)
- 카드1: 선택 단말군 전사 총판매(건) + 전사비중(%)
- 카드2~4: 판매량 기준 본부 Top1~3 (본부명, 판매량, **단말내 비중%** = 본부간점유비)

### 4-3. 비교 하이라이트 배너 3개 (전역 비교 선택값 따라감)
- 배너1: 선택 단말군 총판매 증감(건, %) — 상승 초록 / 하락 빨강
- 배너2: 증가폭 최대 본부 + 증가량
- 배너3: 감소폭 최대 본부 + 감소량
- 전역 `비교`가 "없음"이면 배너 숨김.

### 4-4. 2분할 영역 — 도넛 + 막대
- 좌: `{단말군} 판매 포트폴리오 · 본부간 점유비` 도넛 — slice=10본부, 값=본부별 판매량, 중앙 총건수, 우측 범례(건수·비중% 큰 순)
- 우: `{단말군} 일자별 판매량 분포` 막대 — x=기간 전체 일자, y=전체 본부 합계(=daily_total). **조회일자** 막대는 진한 강조, 나머지 옅게. 막대 클릭 → 조회일자 변경.

### 4-5. 라인차트 3개 (세로 스택, 전체 폭)
공통: x=기간 전체 일자, 라인=10본부(항상 표시, 본부 색), tooltip `mode:index`(같은 날짜 전 본부 동시), 범례 클릭 숨기기/보이기, 안내문 "범례를 클릭하면…".

| 순서 | 제목 | y축 | 정의 |
|---|---|---|---|
| 1 | 일자별 판매량 추이 | 판매건수 | 본부별 해당 단말군 일별 판매량 (`daily`) |
| 2 | 본부내비중 추이 | 본부내비중(%) | 본부 전체판매 중 선택단말군 비율 (`daily_share_in_hq`) |
| 3 | 본부간점유비 추이 | 본부간점유비(%) | 전체본부 선택단말군합 중 해당본부 비율 (`daily_share_of_group`) |

### 4-6. 조회 일자
- 표시줄 "현재 조회 일자: YYYY-MM-DD". 기본=기간 마지막 날.
- 변경: 4-7 표의 날짜 헤더 클릭 또는 4-4 막대 클릭. 순수 클라이언트 상태(막대 강조 + 표 해당 열 하이라이트).

### 4-7. 하단 상세 표 3개 (피벗, 전체 폭, 세로 스택)
공통: 행=10본부(고정 순서 = `HQS`), 열=기간 전체 일자(가로) + 맨 끝 **기간 집계** 열 1개, 본부명 열 sticky, 날짜 헤더 클릭→조회일자 지정. 4-5 라인차트 3개와 **완전 동일 데이터의 표 버전**(순서도 일치).

| 순서 | 표 | 셀 값 | 기간 집계 열 |
|---|---|---|---|
| 1 | 본부별 판매량 (일자별) | 일자별 판매량(건) | 기간합계(건) |
| 2 | 본부별 본부내비중 (일자별) | 일자별 본부내비중(%) | 기간평균(가중, %) |
| 3 | 본부별 본부간점유비 (일자별) | 일자별 본부간점유비(%) | 기간합계 점유비(%) |

## 5. 지표 정의 (매우 중요)

```
전사 총판매(선택 단말군) = 모든 본부의 해당 단말군 판매량 합

본부내비중(%)(본부 j, 날짜 d) = 본부 j의 (선택단말군) 판매량(d) / 본부 j의 (전체 단말 통합) 판매량(d) × 100
본부간점유비(%)(본부 j, 날짜 d) = 본부 j의 (선택단말군) 판매량(d) / 전체 본부의 (선택단말군) 판매량 합(d) × 100

기간합계(건) = 조회 기간 내 일별 판매량의 합
기간평균 본부내비중(%) = Σd(본부 j 단말군)/ Σd(본부 j 전체단말) × 100   (가중평균)
기간합계 본부간점유비(%) = Σd(본부 j 단말군) / Σd(전체본부 단말군) × 100
```

- **본부내비중** 분모 = 그 본부의 전체 단말 판매 → "이 본부에서 파는 폰 중 이 단말군 비율"
- **본부간점유비** 분모 = 전체 본부의 선택 단말군 판매 → "이 단말군을 어느 본부가 많이 팔았나"
- 워딩 고정(과거 "본부내비율"/"전사비중" 모호 표현 폐기). 앱 전역 워딩(v3.5 본부내비중/본부간점유비)과 일치.
