# MNO SYNAPSE — 대시보드 디자인 가이드

> `mno-device-sales`(단말 판매 대시보드)의 디자인 시스템. 본사 임원/팀장용 데이터 대시보드 톤.
> **라이트/다크 둘 다 지원**, CSS 변수로 토큰화. 다른 Colab 대시보드에서도 이 토큰·규칙을 그대로 쓰면 통일감이 남.
> 원본 참조: `~/Downloads/design_guide1.html` (SYNAPSE), 실제 구현: `frontend/index.html`의 `<style>`.

---

## 1. 디자인 철학 (5원칙)
1. **색은 "문제·포커스"에만** — 브랜드 보라/상태색은 강조가 필요한 곳에만. 배경·카드는 무채색 유지.
2. **near-white 중립 캔버스** — 흰 배경 대신 살짝 톤 낮춘 `#FAFBFD`로 눈 편하게, 카드는 순백.
3. **수치는 모노스페이스** — 숫자·메타는 JetBrains Mono + `tabular-nums`로 정렬감.
4. **장식 최소** — 카드 좌측 컬러바 금지, 장식 이모지 자제, 그림자·라운드는 은은하게.
5. **라이트/다크 동등** — 모든 색을 CSS 변수로. 하드코딩 hex 금지(카테고리 색 예외).

## 2. 컬러 토큰 (CSS 변수)
> 컴포넌트는 **항상 `var(--토큰)`** 사용. `[data-theme="dark"]`가 값만 교체.

### 뉴트럴 / 배경
| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `--bg` | `#FAFBFD` | `#0A0B16` | 페이지 배경(near-white/near-black) |
| `--surface` | `#FFFFFF` | `#11132A` | 카드·패널 면 |
| `--surface2` | `#F1F3F7` | `#161A38` | 서브 면(트랙·hover·태그) |
| `--border` | `#E2E6EF` | `#222845` | 테두리 |
| `--line2` | `#CDD3E0` | `#313863` | 보조선·구분선 |

### 텍스트 (4단계 위계)
| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `--text` | `#16182B` | `#EEEFFA` | 본문·핵심 수치 |
| `--t1` | `#3C4060` | `#C2C6E0` | 준강조 |
| `--muted` | `#6E7499` | `#8C92B8` | 라벨·설명 |
| `--dim` | `#9AA0C0` | `#5E6388` | 최약(캡션·플레이스홀더) |

### 브랜드 (SKT 보라 — 포커스 전용)
| 토큰 | 라이트 | 다크 | 용도 |
|---|---|---|---|
| `--accent` | `#3617CE` | `#7E68FF` | 브랜드 포인트(활성탭·선택·강조) |
| `--brand2` | `#5B3FE8` | `#9E8CFF` | 보조 보라(그라데이션·차트) |
| `--accent-soft` | `#E7E2FB` | `#231C58` | 활성 칩 **면**(연한 보라, 꽉 채움 X) |
| `--accent-rgb` | `54,23,206` | `126,104,255` | `rgba()` 조합용 |

### 상태색 (의미 고정 — 남용 금지)
| 의미 | 토큰 | 라이트 | 다크 | soft(배경) |
|---|---|---|---|---|
| 긴급/문제/감소 | `--red` | `#EA002C`(SKT 시그널레드) | `#FF5A5A` | `--red-soft` |
| 정상/과다/증가 | `--green` | `#0E9F6E` | `#2FD79B` | `--green-soft` |
| 주의 | `--warn` | `#F59E0B` | `#FBBF24` | `--warn-soft` |

- 그림자: `--shadow` (라이트=은은, 다크=깊게). 라운드: 카드 12px / 칩 20px / 버튼 6~7px.

## 3. 카테고리 색 (차트 계열)
> ⚠️ **무지개 금지 원칙의 예외 구역.** 카테고리는 고정 매핑 hex(테마별)로 관리, 임의 변경 금지.

### 단말군 색 — **볼륨 가중형**
판매량 큰 군은 **선명**, 작은 군은 **톤다운**. 상태색(빨/초/앰버)과 충돌 회피. SIMonly·기타는 중립 회색.
| 단말군 | 라이트 | 성격 |
|---|---|---|
| S26 | `#4374C4` | 블루(상위, 살짝 톤다운) |
| S25 | `#6E9BD8` | 라이트블루 |
| IP17 | `#8360CC` | 바이올렛(상위) |
| IP16 | `#A98BDB` | 라이트바이올렛 |
| Foldable7 | `#C05B94` | 로즈(핑크 순화) |
| Quantum6 | `#5CA8A0` | 소프트틸 |
| Wide | `#0891B2` | 시안 |
| A17(A17/16) | `#CB7B5B` | 테라코타(웜톤=블루 밸런스) |
| StyleFolder2 | `#9E8FC9` | 라벤더 |
| SIMonly | `#8C93A8` | 중립 회색 |
| Etc | `#C3C8D4` | 밝은 회색 |
- 다크는 각 색을 밝게 보정한 짝(`GCOLOR_DARK`). 매핑 함수 `assignColors()`, 미지 카테고리는 fallback.

### 본부 색 (dot·라인차트)
장식용 카테고리 팔레트(`HQPAL`, HQS 순서 인덱스): `#3B82F6 #16A66B #EF9A1E #8B5CF6 #EC4899 #EF4444 #0EA5A5 #F97316 #22C55E #6366F1 …`
- 히트맵·범례 dot·라인차트 라인에 사용. 상태색과 겹치는 톤은 주의.

## 4. 타이포그래피
- **본문**: `Pretendard Variable, Pretendard, Apple SD Gothic Neo, -apple-system` / 14px / line-height 1.55 / antialiased.
- **수치·메타**: `JetBrains Mono` + `font-variant-numeric: tabular-nums` — KPI 값, 표의 숫자 열, 델타칩.
- 위계: 라벨 11~12px `--muted`(대문자·letter-spacing) / 값 큰 숫자 굵게 `--text` / 캡션 `--dim`.

---

## 5. 컴포넌트 패턴
| 컴포넌트 | 규칙 |
|---|---|
| **카드/패널** | `background:var(--surface)`, border 1px, radius 12px, `--shadow`. **좌측 컬러바 금지.** |
| **KPI 카드** | 라벨(`--muted`) → 값(큰 숫자, mono) → 서브(`--dim`). 앵커 카드만 `--accent-soft` 테두리 강조. |
| **델타칩** | ▲ 증가=`--green` / ▼ 감소=`--red` / – 보합=`--dim`. "▲ +5.7% (+69건)" 형식. |
| **칩(필터)** | 비활성=투명+`--muted`. **활성=`--accent-soft` 면 + `--accent` 텍스트**(꽉 채우지 않음). |
| **상태 dot** | ok=`--green` / mock=`--orange` / err=`--red`, 8px 원. |
| **도넛** | SVG `stroke-dasharray` 트릭(원둘레≈100). 카테고리 색 stroke, 중앙에 총계. |
| **라인차트** | SVG polyline + 마커, index-mode 툴팁(세로 크로스헤어+밴드), 범례 클릭 토글. |
| **표(xtab)** | 숫자 우측정렬·mono, 첫 열 좌측정렬. 필요시 첫 열 `sticky`, 강조 열 `--accent-soft`. |
| **배지/태그** | 레벨 배지는 상태 soft 배경 + 상태색 텍스트. |

## 6. 레이아웃
- **상단 네비**(`position:sticky; top:0; z:100`): 로고 마크(보라 그라데이션 사각형) + 타이틀 + 우측 테마토글·도움말.
- **탭 바**(`sticky; top:50px`): 활성 탭 = `--accent` 텍스트 + 하단 2px `--accent` 보더.
- **전역 컨트롤바**(`sticky; top:91px`): 날짜·비교·필터를 한 줄. 라벨은 11px 대문자 `--dim`.
- **패널 그리드**: 2분할은 `.grid-2 { display:grid; grid-template-columns:1fr 1fr; gap:18px }`.
  - ⚠️ grid 셀 안에 넓은 콘텐츠(스크롤 SVG/표) 넣을 땐 셀에 `min-width:0` (트랙 blowout 방지).
- **가로 넘침**: 넓은 표/차트는 자체 `overflow-x:auto` 컨테이너로 스크롤(페이지 본문은 가로 스크롤 금지).

## 7. 해도 되는 것 / 하면 안 되는 것
✅ **DO**
- 색은 `var(--토큰)`으로만. 상태(빨/초/앰버)는 의미가 맞을 때만.
- 강조는 `--accent-soft`(면)+`--accent`(텍스트) 조합으로 은은하게.
- 숫자는 mono + tabular-nums + 천단위 콤마.
- 라이트/다크 둘 다 확인.

🚫 **DON'T**
- 카드 좌측 컬러바 / 원색 배경 블록 남발.
- 장식용 이모지(📊🚀 등) 남용.
- 하드코딩 hex(카테고리 색 매핑 제외).
- 활성 칩을 브랜드색 **꽉 채움**(soft 면만).
- 카테고리(단말군/본부) 색을 임의로 되돌리기.

## 8. 라이트/다크 테마 구현
- `<body>`(또는 `:root`)에 `data-theme` 속성. `:root`=라이트 기본, `[data-theme="dark"]`가 변수 값만 교체.
- 토글은 `localStorage` persist. 전환 시 `transition: background/color .2s`.
- 컴포넌트는 절대 hex를 직접 쓰지 않으므로 **변수 교체만으로 전체 테마 전환**.

## 9. 화면 워터마크(참고)
- 뷰어 사번+이름을 대각선 반투명 타일로 전 화면 오버레이(유출 억제). 중간톤 회색 `rgba(130,130,150,0.20)`로 라이트·다크 공통.
- 구현법: `docs/watermark-howto.md`.

## 요약 (한 장)
- **캔버스** near-white(`--bg`), 카드 순백(`--surface`) · **텍스트** 4단계(`--text/t1/muted/dim`)
- **브랜드** SKT 보라(`--accent`)는 포커스 전용, 강조는 `--accent-soft` 면
- **상태** 빨=문제/감소, 초=정상/증가, 앰버=주의 (의미 고정)
- **카테고리** 단말군=볼륨 가중, 본부=HQPAL (고정 매핑)
- **폰트** Pretendard, 숫자는 JetBrains Mono + tabular-nums
- **금지** 컬러바·이모지·하드코딩hex·원색 남발
